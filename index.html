Arquitectura del Sistema

Flujo de Ejecución

11 pasos secuenciales que garantizan la correcta transformación de datos crudos a HTML final sin alterar la estructura original de cada componente.

01
Inicialización
Todo List — Estado de Ejecución
El sistema inicializa una lista de tareas con los 7 hitos del proceso. Esto garantiza trazabilidad completa y permite pausar o retomar en cualquier punto del flujo.
manage_todo_list write operation 7 hitos
02
Input — Selección
Solicitar Tipo de Programa
El sistema solicita al usuario el programa activo. Cada programa determina el esquema de datos, los campos requeridos y la plantilla HTML específica a utilizar.
PCO FICOHSA AUSTRO TYP BANORTE DAVIBANK COOPENAE BAC
03
Input — Tipo de Producto
Solicitar Tipo de Producto
Con el programa definido, se selecciona el tipo de producto. La combinación Programa × Producto determina exactamente qué campos se solicitan y cuál plantilla HTML se usa.
✈ Vuelos 🏨 Hoteles 🎯 Promos
04
Input — Datos del Producto
Solicitar y Extraer Datos Estructurados
El sistema presenta el formato exacto de entrada para el par Programa×Producto seleccionado. Los datos se ingresan en formato de parámetros numerados (campo1) y el sistema extrae cada valor automáticamente.

🔑 Clave: Conservación de Estructura

El sistema extrae únicamente los valores de los campos. La estructura HTML del componente permanece intacta — nunca se modifican los contenedores, clases CSS, atributos de accesibilidad ni el orden del markup. Solo se reemplazan los placeholders (campo1) por valores reales.

enlace title (SEO) alt (SEO) pais precio / puntos aerolinea / hotel salida estrellas / descuento
04.5
Transformación
Normalización de Formatos Numéricos
Antes de clasificar, se normalizan todos los valores numéricos. Esto garantiza consistencia visual en el HTML final independientemente del formato de entrada del usuario.
Puntos: 80.000 Precios: $3.091.045 USD: $326
05
Lógica de Clasificación
Clasificar Productos: Nacional / Internacional
Cada producto se clasifica automáticamente según el campo pais. Los destinos dentro de Colombia son nacionales; todo lo demás es internacional. Esta clasificación define la sección del componente donde aparecerá el item.
Internacional → Panamá, Miami, Madrid... Nacional → Bogotá, Cartagena, Medellín...
06
Ordenamiento
Ordenar por Precio Ascendente
Dentro de cada categoría (internacional / nacional), los productos se ordenan de menor a mayor precio. Esto garantiza que el slider HTML presente siempre las mejores ofertas primero.
1° Internacionales ↑ precio 2° Nacionales ↑ precio
07
Validación Intermedia
Mostrar Datos Ordenados al Usuario
Antes de generar el HTML, el sistema muestra los datos clasificados y ordenados para que el usuario los valide. Los campos se numeran secuencialmente (1, 2, 3...) respetando el orden internacional→nacional. Esta es la última oportunidad de corregir valores antes del render HTML final.
Checkpoint de Calidad Numeración Secuencial
08
Selección de Plantilla
Seleccionar Plantilla HTML por Programa × Producto
El sistema selecciona la plantilla exacta correspondiente al par Programa×Producto. Cada plantilla es un componente div.slider-slide con estructura HTML fija, clases CSS, atributos SEO y microdatos precisos para ese programa.

⚡ Actualización Sin Rompimiento de Estructura

La plantilla HTML es inmutable. Solo los valores entre paréntesis (enlace1) se reemplazan. Las clases, el orden del DOM, los atributos ARIA y los meta-atributos SEO nunca cambian, garantizando compatibilidad total con el sitio destino.

09
Generación HTML
Reemplazar Valores en Plantilla
Por cada producto en la lista ordenada, el sistema instancia un bloque de componente y reemplaza sistemáticamente todos los placeholders numerados. Los campos title y alt se colocan en los atributos HTML correctos garantizando SEO óptimo.
title → <a title=""> alt → <img alt=""> href → enlace src → enlaceTipoDeAerolinea
10
Validación de Datos
Confirmar Datos Ordenados
Verificación final de los datos antes de la entrega. El sistema confirma que todos los campos fueron reemplazados, que la numeración es correcta y que no quedan placeholders sin resolver.
Sin placeholders vacíos Numeración correcta
11
Output Final
Entregar HTML Final
El sistema entrega exclusivamente el código HTML final — bloques <div class="slider-slide"> con todos los valores reemplazados. Sin texto adicional, sin comentarios, sin markdown. HTML puro listo para pegarse en el CMS del sitio.
Solo bloques slider-slide Sin texto extra Ready para CMS

Estructura de Componentes

Anatomía del Componente

Cada componente slider-slide es un bloque autónomo con SEO y performance integrados. La actualización reemplaza únicamente los valores, preservando toda la estructura.

🔗
Enlace Principal con SEO
El anchor principal del componente lleva el atributo title que describe el vuelo/hotel completo. Este título está optimizado para SEO con destino, aerolínea, fecha y precio.
  • title = "Destino - Origen - Aerolínea - Fecha - Precio"
  • Descripción semántica completa para crawlers
  • Nunca se modifica el anchor tag — solo su contenido
  • El href toma el valor de (enlace) sin transformar
🖼️
Imagen con Alt Text Optimizado
La imagen del destino o aerolínea incluye un alt redactado profesionalmente con keywords SEO relevantes: nombre del lugar, tipo de producto, precio y propuesta de valor.
  • alt describe la imagen con keywords de conversión
  • Formato: "Destino: descripción con precio y CTA"
  • loading="lazy" preservado para Core Web Vitals
  • Dimensiones width/height nunca alteradas
✈️
Imagen de Aerolínea con Fallback
Para vuelos, la imagen de la aerolínea se inyecta mediante enlaceTipoDeAerolinea. El src apunta a la URL del logo oficial de la aerolínea. El atributo alt descriptivo siempre se mantiene.
  • src = URL imagen aerolínea (no se modifica)
  • Solo aplicable a programas con campo aerolinea
  • Estructura del img tag permanece intacta
  • Formato de imagen preservado según especificación
💰
Datos de Precio y Puntos
Los valores numéricos se normalizan antes de inyectarse en el HTML. Puntos con separador de miles por punto, precios con símbolo y formato local. Los spans contenedores nunca cambian.
  • Puntos: formato 1.136.982 (punto = miles)
  • USD: formato $326 (sin decimales)
  • COP: formato $1.136.982 (punto = miles)
  • Spans y clases CSS de precio intactos
🏨
Metadata de Hotel
Los componentes de hotel incluyen campos adicionales: estrellas (★★★★★), descuento porcentual, nombre del hotel y fechas de compra. Cada campo va al elemento HTML exacto definido en la plantilla.
  • estrellas → elemento de rating con unicode ★
  • descuento → badge de oferta con formato XX%
  • hotel → nombre completo del establecimiento
  • compra → rango de fechas vigentes
📍
País y Ciudad de Destino
El campo pais cumple doble función: es el valor visible en el componente Y es el criterio de clasificación nacional/internacional. Se inyecta sin transformar al elemento correspondiente.
  • Determina clasificación nacional vs. internacional
  • Aparece como etiqueta de destino en el componente
  • También usado en campo pais de vuelos PCO/DAVIBANK
  • Ciudad de salida va en campo separado (salida)

Fórmula del Alt Text SEO 
[Tipo] en [Ciudad]:
[descripción del producto] mostrando
[imagen descriptiva], con [oferta]
y tarifas desde [precio]. Ideal para
[propósito de viaje] en [destino].
Este formato garantiza que el alt text sea útil para lectores de pantalla (accesibilidad), descriptivo para bots de búsqueda (SEO) y persuasivo para usuarios (conversión).

Normalización de Formatos

El sistema normaliza automáticamente todos los valores numéricos antes de inyectarlos al HTML, garantizando consistencia visual sin importar el formato de entrada.

Tipo de Valor Formato Entrada Formato Salida Regla
Puntos 80,000 o 80.000 → 80.000 Punto como separador de miles
Precios COP $3,091,045 o $3.091.045 → $3.091.045 Punto = miles, siempre con $
Acumulación / Redención 2,975 o 2.975 → 2.975 Punto como separador de miles
USD 326.00 o 326,00 → $326 Sin decimales, con símbolo $
Estrellas 4 o ★★★★ → ★★★★ Unicode ★ repetido N veces
Descuento 57 o 57% → 57% Siempre con símbolo %
